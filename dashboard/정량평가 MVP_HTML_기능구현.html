1. MVP 목표
시스템명기자 성과 측정 시스템
목적기자별 기사 성과를 자동 집계하여 객관적인 데이터 제공
대상 사용자편집국 관리자 (평가자), 기자 (조회)
데이터 소스Firebase Storage - daily 폴더 XML (기존 시스템에서 매일 자동 생성)
🎯 1단계 핵심 목표
  • 기자별로 몇 페이지(면)에, 몇 매(건)로 나갔는지 집계
  • 일별 → 주별 → 월별 통계 자동 생성
  • 글자수(총/평균) 자동 계산
  • 엑셀 다운로드 기능
단계별 로드맵
단계내용상세상태
1단계 기사 집계 기자별/면별 기사수, 글자수 자동 집계 (일별/주별/월별) 현재
2단계 가점 입력 특정 기사에 단독/기획 등 가점 체크 (선택사항) 추후
3단계 점수화 정량점수 계산, 상대평가 변환 추후

6. 개발 범위
1단계 (현재)
  • Firebase XML 읽기 / S3 저장
  • XML 파싱 → 데이터 추출
  • 기자별/면별 기사수 집계
  • 일별/주별/월별 통계
  • 기사 목록 조회
  • 엑셀 다운로드
2~3단계 (추후)
  • 개별 기사 가점 입력 (단독/기획)
  • 정량점수 자동 계산
  • 상대평가 변환
  • 조회수 연동
7. 기술 스택
구분기술용도
데이터 소스Firebase Storage → Amazon S3XML 데이터 수집 및 저장
데이터 처리AWS Lambda (Python)XML 파싱, 집계 처리
데이터베이스Amazon RDS (PostgreSQL)기사/집계 데이터 저장
웹 서버Amazon EC2 + FlaskAPI 서버, 웹 대시보드
프론트엔드HTML/CSS/JS + Bootstrap대시보드 UI
8. 로컬 테스트 방법
파일 구조
├── run.bat # 실행 스크립트 (더블클릭) ├── requirements.txt # Python 패키지 ├── src/ │ ├── app.py # Flask 백엔드 서버 │ └── sample_data/ │ └── 2024-10-27.xml # 샘플 XML 데이터 └── dashboard/ └── app.html # 대시보드 UI
실행 방법
# 1. 배치 파일 실행 (Windows) run.bat 더블클릭 # 2. 또는 직접 실행 pip install flask flask-cors python src/app.py # 3. 브라우저 접속 http://localhost:5000
주요 기능
XML 새로고침버튼 클릭 시 src/sample_data/*.xml에서 데이터 다시 로드
기사 목록XML에서 파싱한 기사 목록 (제목, 기자, 면, 글자수)
기자별 통계면별 기사수, 글자수 자동 집계
엑셀 다운로드집계 데이터 엑셀 파일로 다운로드
테스트 방법: Firebase에서 XML 파일을 다운로드하여 src/sample_data/ 폴더에 넣고 "XML 새로고침" 버튼을 클릭하면 실제 데이터로 테스트할 수 있습니다.
